FEATURE OVERVIEW:
=================
Admin can now create and manage daily quest templates for each day of the week.
Each day (Monday through Sunday) can have a unique quest that users will receive
automatically. Templates can be easily created, viewed, and edited.

HOW TO USE:
===========
1. Login as admin
2. Go to Admin Dashboard
3. Click "Gamification" → "Quest Templates" in sidebar
4. See grid of 7 days (Monday through Sunday)
5. Click "Edit" on any day
6. Fill in quest details:
   - Title (e.g., "Complete 3 Lessons")
   - Type (from dropdown)
   - Description (explanation)
   - Target (number to achieve)
   - Reward XP and Coins
7. Watch the preview update as you type
8. Click "Save Template"
9. Template is saved for that day

KEY METHODS:
============

DailyQuest::getByDayOfWeek($dayOfWeek)
- Gets template for specific day
- Parameters: $dayOfWeek (0-6, where 0=Monday, 6=Sunday)
- Returns: DailyQuest model or null

DailyQuest::getWeeklyTemplates()
- Gets all 7 day templates organized
- Returns: Array with day numbers, names, and quest objects

$quest->getDayName()
- Gets human-readable day name
- Returns: String (e.g., "Monday", "Tuesday")

ROUTES:
=======
GET  /admin/daily-quests/templates
     Show all 7 day templates (route: admin.quests.templates)

GET  /admin/daily-quests/templates/{dayOfWeek}/edit
     Show edit form for specific day (route: admin.quests.template.edit)
     Parameters: dayOfWeek (0-6)

PUT  /admin/daily-quests/templates/{dayOfWeek}
     Update template for specific day (route: admin.quests.template.update)
     Parameters: dayOfWeek (0-6)

FORM FIELDS:
============
All fields have validation:
- title (required, max 255 chars)
- description (required, max 500 chars)
- type (required, must be valid type)
- target (required, min 1)
- reward_exp (required, min 0)
- reward_coins (optional, min 0)

DATABASE SCHEMA:
================
ALTER TABLE daily_quests ADD COLUMN day_of_week TINYINT NULLABLE
COMMENT '0=Monday, 1=Tuesday, 2=Wednesday, 3=Thursday, 4=Friday, 5=Saturday, 6=Sunday'

TROUBLESHOOTING:
================
Q: Templates page not found (404)
A: Ensure migration has run. Check routes with: php artisan route:list | grep quests.template

Q: Edit form shows 404
A: Verify day_of_week parameter is 0-6. Check controller method logic.

Q: Live preview not updating
A: Check browser console for JavaScript errors. Verify all input IDs match JavaScript references.

Q: Cannot save template
A: Check form validation. Verify all required fields are filled. Check database column exists.
